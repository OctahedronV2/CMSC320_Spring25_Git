Q1: Why might we consider using Git?

Answer: Because it allows us to easily share our code, but also use version control to be able to undo critical mistakes or revert changes.

Q3: What is the difference between Merging and Rebasing?

Answer: Git merge will preserve the history of both separate branches, while rebasing will create one linear history. Rebasing can be problematic since it changes the commit history, but it does simplify the commit history.
